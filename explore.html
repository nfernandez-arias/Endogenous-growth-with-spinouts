Main analysis: cataloguing spinoffs¶

Preparing dataset¶

Now that we have our data ready, we are ready to analyze the connections between companies in our dataset. I will use the package Networkx, which impelements graphs in Python. To be able to build the graph, we need to have the data organized in such a way that I can easily check whether two firm id's are related by a worker flow. The simplest way to do this is to do a groupby on the key person_object_id, and then use transform and join back into the original dataset, to add a column that has the same values as curr_firm_id, but shifted forward one.

# Restrict analysis to a particular state/industry sub-sample
data = data[(data['curr_category_code'] == 'biotech') & (data['state_code'] == 'WA')]

# Construct directed graph object
G0 = nx.DiGraph()

# Add nodes from the Numpy array 
# data['curr_firm_id'].unique(), which 
# lists each firm in the (restricted) dataset exactly once

G0.add_nodes_from(data['curr_firm_id'].unique())

# Add firm names as attributes (for later)
attributes = {id:name for id,name in zip(data.curr_firm_id,data.name)}
nx.set_node_attributes(G0,'name',attributes)

# Want to draw edge from firm 2 to firm 1 if 
# (1) someone worked at 1 immediately prior to working at 2, 
# and (2) both firms are in the same industry.
# We are already know that curr_category_code is the industry we are interested in,
# so we now simply need to drop observations where previous_category_code is 
# not equal to curr_category_code. Then add an edge corresponding to each remaining
# observation. 

data_sub = data[data['curr_category_code']==data['previous_category_code']]
pairs = data_sub[['curr_firm_id','previous_firm_id']]
pairs = pairs.dropna()
tuples = [tuple(x) for x in pairs.values]
G0.add_edges_from(tuples)
nx.write_gml(G0,'out.gml')
